## 15. 三数之和

给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。

请你返回所有和为 `0` 且 **不重复** 的三元组。

注意：

- 答案中不可以包含重复的三元组
- 你可以按任意顺序返回答案

>示例 1：

- 输入：`nums = [-1,0,1,2,-1,-4]`
- 输出：`[[-1,-1,2],[-1,0,1]]`

>示例 2：

- 输入：`nums = [0,1,1]`
- 输出：`[]`

>示例 3：

- 输入：`nums = [0,0,0]`
- 输出：`[[0,0,0]]`

>提示：

- `3 <= nums.length <= 3000`
- `-10^5 <= nums[i] <= 10^5`

## 题目理解

这题要求的不是“是否存在一个三元组”，而是：

- 找出所有满足和为 `0` 的三元组
- 并且结果不能重复

如果直接三重循环枚举三个位置，时间复杂度会是 `O(n^3)`，在 `n <= 3000` 时通常会超时。

这题的关键优化点有两个：

- 先排序，让数组具有单调性
- 利用双指针或哈希，减少第三层枚举

## 解法对比

### 解法一：排序 + 双指针（推荐）

核心思路：

- 先对数组排序
- 枚举第一个数 `nums[i]`
- 剩下区间里用两个指针 `l`、`r` 去找两个数，使三数之和为 `0`

因为数组有序：

- 如果当前和小于 `0`，说明需要更大的数，所以 `l += 1`
- 如果当前和大于 `0`，说明需要更小的数，所以 `r -= 1`
- 如果刚好等于 `0`，收集答案后继续跳过重复值

为什么排序后适合双指针：

- 固定一个数后，剩余两数之和问题就变成了在有序数组中找目标值
- 双指针可以在线性时间内完成这一步

去重是这题的重点：

- 枚举 `i` 时，如果 `nums[i] == nums[i - 1]`，直接跳过
- 找到一组解后，左右指针都要继续跳过重复元素

复杂度：

- 时间复杂度：`O(n^2)`
- 空间复杂度：`O(1)`（不算排序栈开销时）

### 解法二：排序 + 哈希

也可以先固定第一个数 `nums[i]`，然后把后面的区间当成“两数之和”问题处理：

- 目标变成找到两个数 `b` 和 `c`
- 满足 `b + c = -nums[i]`

做法是：

- 用 `seen` 记录已经遍历过的数
- 当前枚举到 `b` 时，检查 `target - b` 是否已经出现过
- 为了避免重复结果，再额外做一层去重控制

这种写法思路也直观，但在去重处理上通常不如双指针自然。

复杂度：

- 时间复杂度：`O(n^2)`
- 空间复杂度：`O(n)`

## 为什么推荐排序 + 双指针

这题最经典、最稳定的解法就是排序 + 双指针：

- 时间复杂度已经足够优秀
- 利用了排序后的单调性
- 去重规则清晰
- 代码实现也最容易控制边界

对于 `3Sum` 这种“固定一个数，再在剩下区间里找两数”的问题，双指针通常是首选方案。

## 易错点

- 返回的是 **值的组合**，不是下标
- 三元组不能重复，所以外层枚举和内层移动指针时都要去重
- 当 `nums[i] > 0` 时，可以直接结束枚举，因为后面不可能再凑出 `0`
- 找到一个答案后，不能只移动一个指针，通常需要 `l += 1` 和 `r -= 1` 同时进行，再分别跳过重复值
- 排序会改变原数组顺序，如果题目要求保留原顺序就不能这么做，但这题不要求
