## 329. 矩阵中的最长递增路径

给定一个 `m x n` 整数矩阵 `matrix` ，找出其中 **最长递增路径** 的长度。

对于每个单元格，你可以往 `上、下、左、右` 四个方向移动。你 **不能** 在对角线方向上移动，也 **不能** 移动到边界外。

>示例 1：

- 输入：`matrix = [[9,9,4],[6,6,8],[2,1,1]]`
- 输出：`4`
- 解释：最长递增路径为 `[1,2,6,9]`

>示例 2：

- 输入：`matrix = [[3,4,5],[3,2,6],[2,2,1]]`
- 输出：`4`
- 解释：最长递增路径为 `[3,4,5,6]`。注意不允许走对角线。

>示例 3：

- 输入：`matrix = [[1]]`
- 输出：`1`

>提示：

- `m == matrix.length`
- `n == matrix[i].length`
- `1 <= m, n <= 200`
- `0 <= matrix[i][j] <= 2^31 - 1`

## 题目理解

这题表面看是二维网格搜索，本质上可以看成一个 **有向无环图（DAG）**：

- 每个格子是一个节点
- 如果相邻格子的值更大，就从当前格子向它连一条边
- 因为要求“严格递增”，所以不可能形成环

问题就变成了：

- 求 DAG 中的最长路径长度

## 解法对比

### 解法一：DFS + 记忆化搜索（推荐）

核心想法：

- 定义 `dfs(i, j)` 表示“从 `(i, j)` 出发能得到的最长递增路径长度”
- 枚举四个方向，只能走向比当前值更大的格子
- 状态转移：

```python
dfs(i, j) = 1 + max(dfs(ni, nj))
```

其中 `(ni, nj)` 是值更大的相邻格子；如果四周都走不了，答案就是 `1`。

为什么要记忆化：

- 很多路径会重复访问同一个格子
- 不缓存的话会产生大量重复搜索
- 记忆化后，每个格子只会真正计算一次

复杂度：

- 时间复杂度：`O(m * n)`
- 空间复杂度：`O(m * n)`

### 解法二：拓扑排序 + 分层 BFS

把矩阵看成 DAG 后，可以直接做拓扑排序。

常见做法：

- 统计每个点的 `out_degree`，也就是“它能走向多少个更大的邻居”
- `out_degree == 0` 的点就是“峰值”，它们一定是某条递增路径的结尾
- 从这些峰值开始一层层反向剥离，每剥一层，路径长度就加 `1`

为什么层数就是答案：

- 最外层是所有终点
- 再往里一层是距离终点 1 步的点
- 最终剥掉多少层，最长路径就有多少个节点

复杂度：

- 时间复杂度：`O(m * n)`
- 空间复杂度：`O(m * n)`

### 解法三：按数值排序的 DP

还可以把所有格子按数值从小到大排序，再做 DP：

- `dp[i][j]` 表示“以 `(i, j)` 结尾的最长递增路径长度”
- 处理当前格子时，只看四周比它小的格子
- 因为更小的值已经先处理过了，所以可以直接转移

状态转移：

```python
dp[i][j] = max(dp[ni][nj] + 1)
```

其中 `(ni, nj)` 是值更小的相邻格子。

复杂度：

- 时间复杂度：`O(m * n * log(m * n))`
- 空间复杂度：`O(m * n)`

## 为什么推荐记忆化 DFS

这题最自然的写法还是记忆化 DFS：

- 状态定义直观
- 转移关系清晰
- 代码量最少
- 复杂度也已经是最优量级

如果你想从“图论”的角度理解这题，再看拓扑排序会更透彻。

## 易错点

- 必须是 **严格递增**，所以条件是 `>`，不是 `>=`
- 只能走四个方向，不能走对角线
- `dfs(i, j)` 的含义要统一：是“从这里出发”的最长长度，不是“走到这里”的长度
- 空矩阵要先特判
