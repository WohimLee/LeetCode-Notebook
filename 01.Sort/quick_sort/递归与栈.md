是的，**递归和栈本质上可以互相转换**。原因是：**递归本身就是编译器帮你维护的一个“函数调用栈”**。你可以用“显式栈（自己写的 stack）”模拟它，也可以把显式栈改写成递归。

---

# 一、本质：递归 = 隐式栈

当函数递归调用时，系统会自动把信息压入调用栈：

每次递归保存：

* 参数
* 局部变量
* 返回地址
* 当前执行位置

例如：

```python
def f(n):
    if n == 0:
        return
    print(n)
    f(n-1)
```

实际等价于：

```python
stack = [n]
while stack:
    x = stack.pop()
    if x == 0:
        continue
    print(x)
    stack.append(x-1)
```

👉 只是：

递归 → 系统栈
非递归 → 手动栈

---

# 二、递归 → 栈（例：快排）

递归版本：

```python
def quicksort(a, lo, hi):
    if lo >= hi:
        return
    p = partition(a, lo, hi)
    quicksort(a, lo, p-1)
    quicksort(a, p+1, hi)
```

等价栈版本：

```python
def quicksort_iter(a):
    stack = [(0, len(a)-1)]
    
    while stack:
        lo, hi = stack.pop()
        if lo >= hi:
            continue
        
        p = partition(a, lo, hi)
        
        stack.append((lo, p-1))
        stack.append((p+1, hi))
```

---

# 三、栈 → 递归（例：DFS）

栈版本：

```python
stack = [root]
while stack:
    node = stack.pop()
    visit(node)
    stack.extend(children)
```

递归版本：

```python
def dfs(node):
    visit(node)
    for child in children:
        dfs(child)
```

# 四、优缺点对比（非常重要）

| 对比项   | 递归   | 显式栈  |
| ----- | ---- | ---- |
| 代码长度  | 短 ⭐  | 长    |
| 可读性   | 强 ⭐  | 稍复杂  |
| 实现难度  | 简单 ⭐ | 稍难   |
| 空间控制  | 不可控  | 可控 ⭐ |
| 栈溢出风险 | 有 ❌  | 无 ⭐  |
| 性能    | 略慢   | 略快 ⭐ |
| 调试难度  | 较难   | 较易   |

---

# 五、最大区别：栈溢出风险

Python 默认递归深度：

```python
import sys
print(sys.getrecursionlimit())
```

通常：

```python
1000
```

如果递归太深：

```python
RecursionError: maximum recursion depth exceeded
```

例如：

坏快排：

```python
[1,2,3,4,5,6,7,...]
```

递归深度：

O(n)

可能崩溃。

而显式栈不会：

只要内存够就行。

---

# 六、性能区别（实际）

递归慢一点原因：

每次递归都有：

* 函数调用开销
* 保存栈帧
* 恢复栈帧

显式栈只是：

```python
append
pop
```

更轻量。

# 九、一个非常重要理解（核心）

递归：

```
你写逻辑
系统维护栈
```

显式栈：

```
你写逻辑
你维护栈
```

本质完全一样。

# 十、面试经典回答模板（推荐背）

可以直接说：

> 递归本质上依赖系统调用栈保存状态，因此可以用显式栈模拟实现。
> 递归代码更简洁、可读性更好，但存在栈溢出风险和函数调用开销。
> 显式栈实现更安全、可控，适合深度较大的场景，但代码复杂度更高。
