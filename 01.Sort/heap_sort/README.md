## 堆排序（Heap Sort）与快排性能对比

常见选择有两类：

1. **堆排序（Heap Sort）**：完全可以写成非递归，时间复杂度稳定 `O(n log n)`  
2. **TimSort（Python 内置 `sorted()` / `list.sort()`）**：工程实践里通常比手写快排更快（尤其是部分有序数据）


### 1. 为什么选堆排序（而不是继续写一个“非递归快排”）

快排当然也能改成非递归（用显式栈模拟递归），但堆排序有几个明显优点：

- **天然非递归好写**（建堆 + 下沉全是循环）
- **最坏时间复杂度稳定是 `O(n log n)`**
- **原地排序**（额外空间 `O(1)`，不算返回新列表的拷贝）
- 不会像普通快排那样在糟糕 pivot 下退化到 `O(n^2)`

缺点也要诚实说：

- 常数因子通常比优化后的快排大
- CPU 缓存友好性不如快排
- 实际运行中经常比高质量快排慢一些

所以：

- 如果你要 **面试/学习非递归排序**：堆排序很合适
- 如果你要 **Python 实战性能**：优先 `sorted()`（TimSort）

### 2. 堆排序核心思想（升序版本）

升序堆排序使用 **大顶堆**：

- 堆顶始终是当前最大值
- 把堆顶和数组末尾交换
- 缩小堆范围
- 对新的堆顶执行“下沉”（`sift_down`）
- 重复直到只剩一个元素

#### 2.1 例子

数组：

```text
[5, 1, 8, 3, 2]
```

建成大顶堆后（形态可能不唯一）：

```text
[8, 3, 5, 1, 2]
```

然后：

1. 把 `8` 交换到末尾
2. 剩余前 4 个元素重新下沉维护大顶堆
3. 再取出最大值放到倒数第二个位置
4. 重复执行

最后得到升序数组。

### 3. 降序为什么也能复用同一套代码

在 `heap_sort.py` 中，升序/降序不是写两套算法，而是切换堆类型：

- 升序：用 **大顶堆**
- 降序：用 **小顶堆**

代码里通过 `reverse` 参数和 `_heap_should_swap()` 来统一比较规则：

```python
return child < parent if reverse else child > parent
```

这样：

- `reverse=False` 时，维护大顶堆（升序）
- `reverse=True` 时，维护小顶堆（降序）

主流程（建堆、交换、下沉）完全复用。

### 4. 这份实现的结构说明（`heap_sort.py`）

#### 4.1 对外接口

- `heap_sort(nums, reverse=False)`：通用入口
- `heap_sort_asc(nums)`：升序
- `heap_sort_desc(nums)`：降序

特点：

- **返回新列表**
- **不修改原数组**

#### 4.2 `_build_heap(arr, reverse)`

自底向上建堆：

- 从最后一个非叶子节点开始
- 逐个执行 `_sift_down`

最后一个非叶子节点下标公式：

```text
(n - 2) // 2
```

#### 4.3 `_sift_down(arr, start, size, reverse)`（核心）

作用：

- 让 `start` 位置元素向下调整
- 恢复堆性质

逻辑：

1. 找左孩子、右孩子
2. 选出“更适合作为父节点”的那个孩子
3. 若需要交换，则交换并继续向下
4. 否则结束

这是堆排序中最关键的循环。

### 5. 时间复杂度 / 空间复杂度

#### 时间复杂度

- 建堆：`O(n)`
- 取出元素并下沉 `n-1` 次：`O(n log n)`
- 总体：`O(n log n)`

#### 空间复杂度

- 算法本身原地：`O(1)`
- 本实现为了“不修改原数组”先复制一份数据，因此接口层额外用了 `O(n)` 拷贝空间

如果你想要“完全原地”的版本，可以把接口改成直接在传入数组上排序。

### 6. 和快排的对比（重点）

#### 6.1 理论复杂度对比

- 快排平均：`O(n log n)`，最坏 `O(n^2)`
- 堆排稳定：`O(n log n)`（最坏也不退化）

所以如果你非常在意最坏情况，堆排更稳。

#### 6.2 实际性能对比（Python 手写代码）

在 Python 里，手写堆排/快排都通常比不过内置排序，因为：

- `sorted()` / `list.sort()` 是 C 实现
- 算法是 **TimSort**
- 针对真实数据模式做了很多优化（尤其“部分有序”）

也就是说：

- **想学算法**：写快排/堆排
- **想跑得快**：优先 `sorted()`

### 7. 那“比快排性能更好”的非递归方法到底是什么？

如果从 **Python 实战** 角度回答：

- 是 **`sorted()` / `list.sort()`（TimSort）**

它通常会比你手写的快排、堆排都快，且稳定、可靠。

如果从 **算法课/面试实现** 角度回答：

- **堆排序** 是一个很好的非递归替代方案
- **非递归快排** 也可以做，但最坏情况问题仍在（除非再做随机化/三路划分/Introsort）

### 8. 使用示例

```python
from heap_sort import heap_sort, heap_sort_asc, heap_sort_desc

nums = [5, 1, 8, 3, 2]

print(heap_sort_asc(nums))          # [1, 2, 3, 5, 8]
print(heap_sort_desc(nums))         # [8, 5, 3, 2, 1]
print(heap_sort(nums, reverse=True))
print(nums)                         # 原数组不变
```

### 10. 一句话结论

- **有**不用递归且性能同级的排序：`堆排序（Heap Sort）`
- **在 Python 实战里通常更快**的是：`sorted()` / `list.sort()`（TimSort）
